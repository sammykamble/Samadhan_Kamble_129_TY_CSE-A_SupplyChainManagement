                        VIVA PREPARATION

Key Concepts to Explain:

1. NORMALIZATION
   - Database is in 3NF (Third Normal Form)
   - No transitive dependencies
   - All non-key attributes depend only on primary key

2. CONSTRAINTS
   - PRIMARY KEY: Unique identifier for each record
   - FOREIGN KEY: Maintains referential integrity
   - CHECK: Validates data (rating 1-5, prices > 0)
   - UNIQUE: Prevents duplicates (email, tracking)

3. JOINS
   - INNER JOIN: Returns only matching records from both tables
   - LEFT JOIN: Returns all from left + matching from right

4. VIEWS
   - Pre-defined queries stored in database
   - Simplifies complex queries
   - Provides data abstraction

5. STORED PROCEDURES
   - Pre-compiled SQL code
   - Reusable business logic
   - Better performance
   - Reduces network traffic

6. TRIGGERS
   - Automatic actions on events
   - Used for: inventory updates, logging, validation
   - Fires on INSERT, UPDATE, DELETE

7. INDEXES
   - Speed up query performance
   - Created on foreign keys and frequently queried columns

Expected Questions:

Q: Why use foreign keys?
A: To maintain referential integrity and ensure data consistency.

Q: Difference between INNER JOIN and LEFT JOIN?
A: INNER JOIN returns only matches. LEFT JOIN returns all left records.

Q: What are triggers?
A: Automatic actions that execute when specific events occur.

Q: Purpose of stored procedures?
A: Reusable, pre-compiled SQL code for better performance.

Q: How did you prevent SQL injection?
A: Used mysqli_real_escape_string() and input validation.

                         SAMPLE SQL QUERIES
================================================================================

-- View inventory summary
SELECT * FROM vw_product_inventory_summary;

-- Get low stock products
CALL sp_get_low_stock_products();

-- Check pending orders
SELECT * FROM vw_active_orders;

-- Supplier performance
SELECT s.supplier_name, COUNT(po.order_id) as orders, 
       SUM(po.total_amount) as total_value
FROM suppliers s
LEFT JOIN purchase_orders po ON s.supplier_id = po.supplier_id
GROUP BY s.supplier_id
ORDER BY total_value DESC;

-- Warehouse utilization
SELECT * FROM vw_warehouse_utilization;

-- Products below reorder level
SELECT p.product_name, SUM(i.quantity) as stock, p.reorder_level
FROM products p
LEFT JOIN inventory i ON p.product_id = i.product_id
GROUP BY p.product_id
HAVING stock < p.reorder_level;
